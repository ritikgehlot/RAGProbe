# RAGProbe 🔍

**Is this answer actually in the sources? A meta-evaluated faithfulness audit for RAG systems.**

Most RAG pipelines ship with zero systematic evaluation — "the answer looks right" is the QA process. RAGProbe measures whether answers are *grounded in their source passages*, and — this is the part most eval tools skip — **measures the measurers**: every scorer is benchmarked against provable ground truth, so you know each scorer's real precision, recall, and blind spots before trusting its verdicts.

> Zero-dependency core: `pip install -e .` gives you the eval-set builder, the lexical scorer, and the HTML report with **no third-party packages**. The NLI scorer is an optional extra (`pip install -e ".[nli]"`).

## How ground truth is provable: contamination by construction

1. **Build grounded answers.** Take (passage, question, answer-span) triples from SQuAD v2 (CC BY-SA 4.0) and construct answers from the passage sentences around the true span — support is guaranteed *by construction*.
2. **Inject contamination.** Create corrupted twins with exactly one seeded perturbation each: **entity swap** · **number change** · **negation flip** · **foreign sentence injection**. We know which answers are unfaithful and *which sentence* carries the corruption.
3. **Meta-evaluate scorers.** Because ground truth is known, each scorer gets a real detection report: precision, recall, F1, false-alarm rate on clean answers, recall per contamination type, and sentence-level localization accuracy.

Precision **1.00** · false-alarm rate **0.00** · localization **1.00** (when it fires, it fires on the right sentence).

**The finding, stated honestly:** cheap token-overlap detects *wholesale injection* almost perfectly but is nearly blind to *single-token corruptions* — swapped entities, altered numbers, flipped negations sail through, because one changed word barely moves overlap in a long sentence. Those are exactly the subtle hallucinations that hurt users. That asymmetry is the whole argument for semantic scorers, and RAGProbe makes it measurable instead of anecdotal.

## Quickstart

```bash
pip install -e .                 # zero-dependency core
ragprobe build --n 200 --seed 7                  # downloads SQuAD, builds eval set
ragprobe run --scorers lexical                   # meta-eval + HTML report

pip install -e ".[nli]"          # optional: semantic scorer (torch + transformers)
ragprobe run --scorers lexical,nli               # the interesting comparison

export ANTHROPIC_API_KEY=sk-...  # optional: LLM-judge scorer
ragprobe run --scorers lexical,nli,judge
```

Open `reports/report.html` — a self-contained audit report with a scorer scoreboard, a detection matrix (each scorer's fingerprint across contamination types), and filterable case files showing sentence-level verdicts against the ◈-marked injected corruption.

## Scorers

| Scorer | Mechanism | Cost | Designed strength | Known blind spot |
|---|---|---|---|---|
| `lexical` | weighted token coverage vs passage windows (entities/numbers ×3) | free, stdlib | injected foreign content | single-token edits, negation (pinned by a unit test) |
| `nli` | cross-encoder entailment, P(entailment) per sentence vs top-k premise | local model, ~140 MB | negation, semantic contradiction | paraphrase drift beyond premise window |
| `judge` | LLM verdict per sentence, strict JSON rubric, retries + token accounting | API calls | nuanced claims | needs its own meta-eval before being trusted — which this tool provides |

## Repository layout

```
src/ragprobe/
  dataset.py    SQuAD download, grounded-answer construction, JSONL I/O
  perturb.py    four seeded contamination generators + localization ground truth
  scorers/      base interface · lexical · nli (lazy deps) · judge (optional API)
  runner.py     meta-evaluation: P/R/F1, FAR, per-type recall, localization
  report.py     self-contained HTML audit report (no external JS)
  cli.py        ragprobe build / ragprobe run
tests/          perturbation determinism, scorer behavior, meta math, judge parsing (mocked)
```

## Limitations (read before quoting numbers)

- Constructed answers are near-verbatim; real RAG answers paraphrase. Detection rates here are an **upper bound for extractive-style answers**, not a universal hallucination benchmark.
- Four contamination types ≠ all hallucination modes (no unsupported-inference, no partial-truth blending yet — see roadmap).
- The judge scorer's agreement with humans is not yet measured; until it is, its verdicts deserve the same skepticism as any other scorer's.
